Давай без кода. Обсудим теоретическую часть. Изначально требуется механизм проверки файлов на соответствие определенным условиям.
Надо начать с малого, проверять читаемость фбх файлов и сполнение основных функций.
Дальше распишу полный список задач.
На вход приходит .zip архив. Каждая проверка проходит отдельно и в финальном отчете выдает результат (true, false)
1. Проверяется наименование архива. (есть определенные требования к неймингу, определенные правила с существующим набором заготовленных имен и тд)
2. Проверяется структура файлов. Отсутствие папок. Наличие fbx файла, наличие json файла. Наличие текстур (потребуется так же работа с текстурами но с этим механизмом можно попозже).
3. Проверяется размер файла.
4. Названия файлов внутри архива.
5. Названия объектов геометрии внутри fbx файла
6. Наименование материала (шейдера)
7 Расположение pivot поинта
8. Наличие отдельного файла с источниками света и их наименование
9 Количество треугольников у объекта
10 11Текстуры (сводная таблица с определенным набором проверок текстур, детали позже, главное сам факт возможности проверки)
12 Текстурная плотность (пока без уточнений, но определенная работа с плотностью текстур модели)
13 проверка json файла из архива, так же сводная таблица на соответствие требованиям

Вопрос возможно ли проводить такие проверки с использованием предложенных тобой вариантов, с каким будет проще (FBX SDK звучит заманчиво,тк большинство пользователей и так будут использовать продукты Autodesk для создания проверяемых файлов)
